## 功能介绍

Konnex BOT 是一个自动化机器人，支持以下功能：

### 🎯 主要功能

1. **邀请码注册模式** - 使用邀请码批量注册新账户
2. **正常签到模式** - 自动执行每日签到和任务
3. **混合模式** - 先注册新账户，再执行签到

### ✨ 特色功能

- 🔄 多账户管理
- 🎁 邀请码自动分配
- 🌐 代理支持（可选）
- 📊 详细的运行日志
- ⏰ 24小时自动循环

## 安装步骤

### Windows 系统

1. **安装 Node.js**
   - 下载并安装 [Node.js](https://nodejs.org/) (v16 或更高版本)

2. **下载项目**
   ```bash
   # 解压到任意目录
   cd Konnex-BOT-main
   ```

3. **安装依赖**
   ```bash
   npm install
   ```

### Ubuntu/Linux 系统

1. **安装 Node.js**
   ```bash
   curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
   sudo apt-get install -y nodejs
   ```

2. **下载并安装**
   ```bash
   cd ~/Konnex-BOT-main
   npm install
   ```

## 配置说明

### 📁 文件结构

```
Konnex-BOT-main/
├── bot.js                    # 主程序
├── package.json              # 项目配置
├── accounts.txt              # 正常运行账户（必需）
├── referral_accounts.txt     # 待注册账户（可选）
├── referral.txt              # 邀请码配置（可选）
└── proxy.txt                 # 代理配置（可选）
```

### 1️⃣ accounts.txt - 正常运行账户

存放日常签到的账户私钥，每行一个：

```
0x1234567890abcdef1234567890abcdef1234567890abcdef1234567890abcdef
0xabcdef1234567890abcdef1234567890abcdef1234567890abcdef1234567890
```

**注意事项：**
- 每行一个私钥
- 支持带 `0x` 前缀或不带前缀
- 空行会被自动忽略

### 2️⃣ referral_accounts.txt - 待注册账户

存放需要使用邀请码注册的新账户私钥：

```
0x新账户私钥1
0x新账户私钥2
0x新账户私钥3
```

**使用流程：**
1. 将待注册账户私钥添加到此文件
2. 运行脚本选择"邀请码注册模式"
3. 注册成功后，将私钥移至 `accounts.txt` 进行日常使用
4. 从 `referral_accounts.txt` 删除已注册的私钥

### 3️⃣ referral.txt - 邀请码配置

存放邀请码，每行一个：

```
CODE1
CODE2
CODE3
```

**分配规则：**
- 第1个账户使用第1个邀请码
- 第2个账户使用第2个邀请码
- 如果账户数量多于邀请码，会循环使用
- 以 `#` 开头的行为注释

**示例：**
```
# 我的邀请码
1UL2W8ED
VM73LFJZ
# 朋友的邀请码
FRIEND_CODE
```

### 4️⃣ proxy.txt - 代理配置（可选）

如果需要使用代理，每行一个代理地址：

```
http://username:password@proxy.example.com:8080
socks5://username:password@proxy.example.com:1080
```

**支持的协议：**
- HTTP: `http://`
- HTTPS: `https://`
- SOCKS5: `socks5://`

**注意事项：**
- 如果不需要代理，保持文件为空或删除所有代理行
- 代理按顺序分配给账户（第1个账户用第1个代理）
- 如果代理数量少于账户数量，会循环使用

## 使用方法

### 🚀 启动脚本

```bash
npm start
```

### 📋 选择运行模式

启动后会显示菜单：

```
请选择运行模式:

  1. 邀请码注册模式 (待注册账户: X)
  2. 正常签到模式 (账户数: Y)
  3. 两者都执行 (先注册后签到)
  0. 退出

请输入选项 (0-3):
```

### 1️⃣ 邀请码注册模式

**适用场景：** 批量注册新账户

**操作步骤：**
1. 准备好待注册账户的私钥
2. 添加到 `referral_accounts.txt`
3. 配置邀请码到 `referral.txt`
4. 运行脚本，选择 `1`
5. 等待注册完成

**输出示例：**
```
=== 邀请码注册模式 ===
待注册账户数: 3

[ 账户 1 ]
Proxy   : No Proxy
正在注册账户...
Referral: 1UL2W8ED
✓ 连接成功
✓ CSRF 令牌获取成功
✅ 注册成功!
Balance : 0 Points
```

### 2️⃣ 正常签到模式

**适用场景：** 日常自动签到和任务

**操作步骤：**
1. 确保 `accounts.txt` 中有账户私钥
2. 运行脚本，选择 `2`
3. 脚本会自动循环运行

**输出示例：**
```
账户总数: 2

[ 账户 1 ]
Proxy   : No Proxy
正在登录...
✅ 登录成功
Balance : 100 Points
正在签到...
✅ 签到成功
Testnet : 
  Task: Complete 5 transactions
  Status: In Progress
```

**循环运行：**
- 完成所有账户后，等待24小时
- 自动开始下一轮签到
- 按 `Ctrl+C` 可随时停止

### 3️⃣ 混合模式

**适用场景：** 既要注册新账户，又要执行签到

**操作步骤：**
1. 配置好 `referral_accounts.txt` 和 `accounts.txt`
2. 运行脚本，选择 `3`
3. 先完成注册，再执行签到

## 常见问题

### ❓ 如何获取私钥？

使用 MetaMask 等钱包导出私钥：
1. 打开钱包
2. 点击账户详情
3. 导出私钥
4. 复制私钥到配置文件

### ❓ 注册失败怎么办？

**检查步骤：**
1. 确认私钥格式正确
2. 确认邀请码有效
3. 检查网络连接
4. 查看错误日志

**常见错误：**
- `❌ 连接失败` - 网络问题或代理配置错误
- `❌ 获取 CSRF 令牌失败` - API 访问问题
- `❌ 注册失败 - 认证失败` - 私钥或邀请码问题

### ❓ 代理连接失败？

**解决方案：**
1. 检查代理格式是否正确
2. 测试代理是否可用
3. 尝试不使用代理（清空 `proxy.txt`）
4. 联系代理服务商获取正确配置

### ❓ 如何停止脚本？

按 `Ctrl+C` 即可停止运行。

### ❓ 登录失败怎么办？

**可能原因：**
- 私钥错误
- 网络问题
- API 限流

**解决方法：**
1. 检查私钥是否正确
2. 等待几分钟后重试
3. 更换代理或不使用代理

### ❓ Ubuntu 服务器运行报错？

**错误：** `Cannot find module 'readline-sync'`

**解决方案：**
```bash
cd ~/Konnex-BOT-main
npm install
npm start
```

### ❓ 如何查看运行日志？

所有操作都会实时显示在终端，包括：
- 登录状态
- 签到结果
- 任务进度
- 错误信息
